## Вторая лекция.

Например, вот такой конфликт:

На второй лекции проходили ветки, их создание, перемещение между ветками, их удаление и слияние:

* создать ветку (git branche <имя_ветки>);

* посмотреть существующие ветки (git branch);

* переключиться на ветку (git checkout <имя_ветки>);

* слить ветки (git merge <имя_ветки>) - добавляет в ветку, в которой находишься сейчас, ту ветку, имя которой называешь в команде;

* удалить ветку (git branch -d <имя_ветки>).

Иногда при слиянии веток происходит конфликт. Возникает тогда, когда в ветку с более актуальной информацией пытаешься добавить ветку с менее актуальной информацией (см по времени последнего commit).

В этом случае предлагаются варианты:

* Accept Current Change - принять текущую версию;

* Accept Incoming Change - принять поступившую версию;

* Accept Both Change - принять обе версии;

* Compare Chenges - сравнить версии.

## Третья лекция.

На третьей лекции была работа с удаленными репозиториями на GitHub. Для этого в первую очередь нужно создать свой аккаунт на GH и подружить VSC с GH (если возникнут сложности, то git даст подсказку, у меня уже был на тот момент и аккаунт и "дружба").

Для того, чтобы выложить свой локальный репозиторий на GH нужно:

1. В своем аккаунте ищем "+" - new repository - даем ему имя - create repository.

2. GH предлагает три варианта:
   * создать новый репозиторий в терминале;

   * добавить существующий репозиторий;

   * импортировать из другого репозитория.

Наш вариант 2, так как локальный репозиторий уже создан.

3. Копипастим построчно команды, которые предлагает GH в свой терминал. Готово.

Для того, чтобы загрузить новые изменения в файле из локального в удаленный репозиторий GH нужно:

1. Вносим любые необходимые изменения в файл, коммитим.

2. Вызываем в терминале команду, чтобы импортировать: git push.

Для того, чтобы загрузить новые изменений в файле из удаленного репозитория в локальный репозиторий нужно:

1. Внести любые изменения в файл (можно напрямую на GH, либо подгрузить любой репозиторий со сторонних источников).

2. Запускаем в локальном репозитории в терминале команду: git pull (подгружает в том числе и все вестки, которые есть в удаленном репозитории).

OPEN SOURSE:

1. Зайти в интересующий нас репозиторий на GH.

2. Нажимаем Fork, данная команда полностью копирует этот репозиторий в наш аккаунт.

3. Забираем этот репозиторий в наш локальный: git clone <адрес ссылки репозитория>.

4. В локальном репозитории создаем отдельную ветку для изменение, создаем изменения, коммитим.

5. Загрузить свои изменения в свой аккаунт (git push).

6. Кнопка Compare & pull request - предложить изменения (оставляем комментарии, чтобы автору изначального репозитория было понятно что от него хотим или предлагаем - create pull request). Готово
